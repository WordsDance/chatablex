## 一句话

**ChatableX 相信：AI 不该替你做所有事，而该准确调用你已经验证过的工具。**

我们不做「把几百个工具塞进上下文、再让模型猜该用哪个」的黑盒智能体。用户带着明确意图打开对话，从左侧工具商店检索、拖拽本次任务需要的工具，Agent 在隔离、可追踪的会话环境中精准执行 — 确定性业务交给稳定工具，不确定性交给 AI 推理。

## 核心理念：指挥官，而非执行者

当前行业 — OpenAI Codex、Anthropic Claude Code、阿里 Coder、Coze、各类「悟空 AI」— 都在探索同一个问题：**智能体如何真正落地生产？**

大家的共识是「Agent + 工具」。但多数产品把重点放在了「让 AI 做更多」— 自动检索工具、自动写代码、自动编排工作流 — 却普遍忽略了一个生产环境真正在乎的点：

> **工具有没有被准确调用？调用过程能不能追溯？出错了能不能定位？**

我们的观点恰恰相反：

| 层次 | 谁负责 | 要求 |
|------|--------|------|
| **核心业务** | 人 + 稳定工具 | API、脚本、小工具的执行结果必须**经过验证和测试** |
| **编排调度** | AI Agent | 接收你的指令，**准确路由**到对应的工具或 MCP |
| **探索创新** | AI 推理 | 处理模糊意图、规划步骤、解释结果 |

AI 是**指挥官**，不是**全能执行者**。把确定性交给工具，把不确定性交给模型 — 这才是能进生产环境的 Agent 架构。

## 我们如何解决「工具选错」

同一个业务场景，不同用户可能各自制作了成百上千个定制工具。如果全部预加载进 Agent 上下文，或用 RAG 检索后一股脑塞给模型 — 选错工具、参数幻觉、调用链不可控，几乎是必然的。

ChatableX 的做法是 **会话级工具隔离** — 通过对话决定本次任务用哪些工具，而不是让智能体提前加载一大堆：

```
┌─────────────┐     检索 / 筛选      ┌──────────────────┐
│  工具商店    │ ──────────────────▶ │  本次对话工具栏   │
│  (左栏)     │     拖拽装配         │  (输入区标签)     │
└─────────────┘                     └────────┬─────────┘
                                             │ 仅注入已选工具
                                             ▼
                                    ┌──────────────────┐
                                    │  Agent 执行上下文  │
                                    │  小而精，而非大而全 │
                                    └──────────────────┘
```

**四条设计原则：**

1. **意图先行** — 用户要做 PPT、处理表格、发邮件，意图是明确的；在商店里搜、拖、选，而不是让 Agent 从几百个工具里「猜」。
2. **对话隔离** — 内置少量通用基础工具；业务工具由用户按会话主动装配，互不污染。
3. **质量筛选** — 同一业务可能有成千上万个工具，进入对话前需经过验证状态、使用反馈、评分排名等筛选，避免幻觉。
4. **动态加载** — 按需加载、按需卸载；每个对话拥有独立的运行环境，会话结束后依赖自动清理。

## 核心能力

### 1. 拖拽装配 · 个人工具商店

左侧扩展栏是你的**个人工具工作台**：浏览市场、管理已安装扩展、存放自己制作的工具。

普通用户做完一个工具，可以一键存进左侧商店；下次要做同类任务，从商店里检索、拖到对话框里就行 — 不用改配置，不用碰代码。

### 2. Harness Inspector · 工具调用全程可追踪

市面大多数 Agent 只有左侧对话框，工具调用过程是黑盒。ChatableX 在右侧提供 **Harness Inspector** — AI 行为的透明化窗口：

- **推理追踪** — Agent 在想什么，实时可见
- **工具流水线** — 每次工具调用的输入、输出、状态，树状展示
- **环境上下文** — 当前会话已装配的工具、文件状态
- **关键操作确认** — 写文件、调付费接口、发布工具等动作，需用户批准后再继续

左感性（对话结论），右理性（执行细节）。

### 3. 零代码造工具 · 录制即技能

- **描述即创造** — 在对话中表达意图，Agent 生成并迭代可执行工具
- **录制即技能** — 在界面上示范一次操作路径，AI 自动蒸馏为可复用技能，以演示代替编码
- **版本管理** — 工具支持草稿、测试、稳定、归档等阶段，随时回退

### 4. 会话级依赖隔离

每个对话拥有独立的运行环境。不同会话安装的工具依赖互不干扰，对话空闲后自动清理 — 不会出现「上次装的包把这次搞坏了」的情况。

### 5. 先计划、再执行

Agent 先制定执行计划，再逐步调用工具，失败可重试、可换方案、可中止。配合环境反馈，每一步都可定位、可中断、可回溯 — 不是一次性碰运气。

## 典型使用场景

**举例：给图片加水印**

1. 在左侧扩展栏搜索「图片加水印」，拖到输入框
2. 上传图片，用自然语言描述参数（文字内容、字号、位置）
3. 右侧 Inspector 实时展示 Agent 如何解析意图、映射参数、调用工具
4. 处理完成，文件保存到当前会话；工具留在商店，下次直接复用

```
明确任务 → 商店检索 → 拖拽装配 → Agent 规划 → 追踪执行 → 用户确认 → 输出结果
```

## 为谁而生

- **业务专家 / 运营** — 不会写代码，但需要可重复使用的自动化工具
- **中小企业** — 希望 AI 接入已有 API 和小工具，而不是重写整套系统
- **对 Agent 黑盒感到不安的人** — 需要看清每一步推理与工具调用
- **工具创作者** — 制作、测试、沉淀、复用的完整闭环
